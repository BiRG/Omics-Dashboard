Omics Dashboard Core Concepts

Samples

The lowest level of data organization in Omics Dashbaord is the Sample. A Sample represents the results of one run of your analysis and is created from one file. It is possible for one Sample to represent the data for more than one real-world sample. Samples have attributes, which are scalar values which describe the Sample, and datasets, which are arrays of values of the same type describing the experimental results. Typically, a sample will have a dataset called 'x', which represents the independent variable/abscissa, and a dataset called 'Y' which represents the dependent variable/ordinate. There is one universal value for 'x' for all of the Sample, and there will be one row for 'Y' for each real-world sample.

Sample Groups

The Omics Dashboard sample upload process always creates a Sample Group, which is a collection of one or more Samples. Sample Groups exist entirely in the database and do not have their own files.

Collections

The primary unit of data organization in Omics Dashboard is the Collection, which is a single file containing the data from one or more Samples. Collections are typically created from multiple files contained in the same Sample Group, but any combination of Samples may be used to create a Collection. Each row of a collection's 'Y' dataset represents the data for one real-world sample. The creation of Collections from Samples will also result in the creation of 'label columns', which are labels for the rows in 'Y' created from the common attributes of the Samples from which the Collection is created. Label columns may be edited directly from Omics Dashboard. Other datasets are edited via the web-based analysis tool, workflow editor modules, or other Omics Dashboard client software.

Administration

Administrator Accounts

Administrators have the power to delete or edit any record in the system, including other user's accounts. You should limit who becomes an administrator. It may be useful to use a separate email address for your administrator and user accounts to prevent accidental deletion or editing of other user's records.
